# Publishing

Publishing is the release of a reviewed production package, not simply the export of a video file.

## Required Publication Package

- final video export;
- approved title;
- approved description;
- chapters or timestamps;
- captions;
- transcript;
- thumbnail or visual preview;
- links to the canonical episode and repository;
- disclosure that the companion is an AI-created visual presenter;
- publication target metadata;
- review record.

## Metadata Principles

Publication metadata should be clear, accurate and grounded in the episode. It should help viewers understand what the episode covers without overstating certainty or implying that derived media replaces the written journal.

## Traceability

Each publication should record:

- canonical episode path;
- source version or commit;
- production artefact version;
- final export date;
- publication target;
- known unresolved editorial notes, if any.

## Post-Publication Changes

If the canonical episode changes after publication, the production team should decide whether derived media needs an update, correction note, replacement, or no action. The decision should be recorded where future readers can understand it.

# Episode Production Workflow

This workflow describes how a canonical Articulate episode becomes derived media.

## 1. Confirm Source

Identify the canonical Markdown episode and confirm that it is approved for production. Record the source path and version in the production plan.

## 2. Create Production Plan

Use the episode to define the editorial treatment, audience, target duration, medium-specific goals, major sections, companion usage, required visual assets, risks and review gates.

## 3. Draft Narration and Storyboard

Create narration and storyboard drafts from the approved plan. These drafts may be AI-assisted, but they must preserve source meaning and include traceability to source sections.

## 4. Prepare Scene List and Asset Register

Turn the storyboard into an operational scene list and record all required assets in the asset register. Assets should have identifiers before they are generated, recorded or captured.

## 5. Produce Approved Assets

Create or capture companion scenes, diagrams, slides, whiteboard scenes, code demonstrations, screen recordings and supporting imagery. Record versions, sources and review notes.

## 6. Record Human Voice

Record Russell's narration from the approved script. Retakes and delivery changes should be reflected back into the narration script or edit notes.

## 7. Assemble Rough Cut

Combine narration, scenes, assets, subtitles and preliminary metadata into a rough cut. The rough cut is a review artefact, not a publication candidate.

## 8. Review and Revise

Run the human review gates. Revisions should return to the earliest artefact that explains the problem: plan, script, storyboard, scene list, asset register or edit.

## 9. Prepare Publication Package

Prepare final export, transcript, captions, title, description, chapters, thumbnail, source links and companion disclosure.

## 10. Publish

Publish only after final approval. Record publication target, date, final source references and any post-publication follow-up.

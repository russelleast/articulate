# Articulate Companion

The Articulate Companion is an AI-created visual presenter used where a conventional presenter might normally appear on camera.

It is not a digital clone of Russell and must not be presented as one. Russell provides the thinking, authorship, judgement and recorded voice. The companion provides an intentional visual presence that helps guide viewers between narration, diagrams, demonstrations, sketches and other visual material.

## Purpose

The companion gives the video format a consistent presenter presence without making the production dependent on filming a human presenter. It helps orient the viewer, introduce sections, create continuity and make transitions feel deliberate.

## Relationship with Russell

Russell remains the author and human accountable voice of the work. The companion is a production device: a designed visual representative of the Articulate media system. It may appear to present or guide, but it does not originate the architectural argument.

## Role

- introduce the episode and its central question;
- guide transitions between reflective and explanatory sections;
- frame diagrams, sketches and demonstrations;
- provide a calm visual presence during narration-led moments;
- close the episode by returning to the main architectural question.

## Speaking Style

The companion's visual performance should match Russell's recorded narration. It should feel calm, thoughtful, technically literate, restrained and curious. It should not appear to be improvising, arguing independently or adding unsupported claims.

## Visual Role

The companion is most useful when the viewer benefits from a presenter-like presence. It should appear selectively rather than continuously. Diagrams, repository views and written artefacts should take visual priority when they carry the explanation.

## Appropriate Scene Types

- opening and closing scenes;
- section introductions;
- transitions between abstract ideas and concrete artefacts;
- moments where a conventional presenter would help pace the explanation;
- short reflective pauses between dense architectural sections.

## Inappropriate Scene Types

- technical diagrams that need the full frame;
- code or terminal demonstrations where precision matters;
- scenes where companion presence would imply authority beyond the source;
- emotional performance or comic relief;
- any scene that could make the companion seem like the autonomous author.

## Transparency and Disclosure

The companion must be explicitly disclosed as an AI-created visual presenter. It must not be framed as Russell, as a hidden replacement for Russell, or as a real person who independently created the content.

## Accessibility

The companion should not carry essential meaning through gesture or expression alone. Important information must remain available through narration, captions, transcripts or visible text.

## Consistency

The companion should retain recognisable continuity across episodes while allowing visual refinement. Changes to the companion's design direction should be documented so viewers understand that it is a production element rather than a shifting identity.

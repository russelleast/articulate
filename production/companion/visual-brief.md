# Companion Visual Brief

## Design Direction

The companion should sit somewhere between architectural guide, digital researcher, technical presenter and AI-native companion.

The visual language should be clearly artificial without becoming cold, theatrical or deceptive. It should suggest a designed presence for an AI-assisted production system, not a simulated human presenter.

## Desired Impression

- calm and focused;
- intellectually credible;
- comfortable around diagrams, code and architectural concepts;
- restrained rather than performative;
- distinct enough to be recognisable across episodes;
- compatible with journal, architecture and production themes.

## Avoid

- photorealistic human impersonation;
- celebrity, founder or author resemblance;
- cyberpunk styling;
- mascot behaviour;
- exaggerated facial performance;
- glossy corporate training-video aesthetics;
- visual complexity that competes with diagrams.

## Visual Context

The companion should work in:

- simple studio-like presenter frames;
- split attention moments beside title or section text;
- transitions into diagrams;
- calm closing scenes;
- occasional whiteboard or research-desk contexts.

It should not require a fixed fictional environment. The production system should be able to place it beside diagrams, repository views or abstract visual fields without breaking consistency.

## Unresolved Decisions

The following decisions are intentionally unresolved:

- name;
- gender presentation;
- exact appearance;
- wardrobe or surface treatment;
- rendering method;
- vendor;
- degree of stylisation;
- recurring background or environment.

These should be resolved through future design exploration and documented as production decisions when they begin to constrain repeatable work.

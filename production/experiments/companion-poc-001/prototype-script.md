## Segment N002

- Corresponding scene: S002
- Source reference: Introduction
- On-screen text: Understanding the architecture itself
- Emphasis: "not the tools" and "architecture itself".
- Pause notes: small pause before the question fragments appear.

Narration:

> Modern technology is remarkably capable. Cloud platforms scale, frameworks mature, infrastructure can be provisioned quickly, and AI can already help with many development tasks. Yet architects still return to the same questions. What changed? Why was it designed this way? Can we trust the documentation?

## Segment N003

- Corresponding scene: S003
- Source reference: The Reality of Modern Architecture
- On-screen text: Part of the truth
- Emphasis: "scattered" and "whole picture".
- Pause notes: half-beat after "people's memories".

Narration:

> The reason is that architectural knowledge is scattered. Some of it lives in source code. Some in Infrastructure as Code. Some in ADRs, diagrams, slide decks, monitoring platforms and people's memories. Each source contains part of the truth. None contains the whole picture.

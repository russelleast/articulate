# Background Scene Prompts

Purpose: explore visual contexts for source-grounded non-slide sequences.

## Fragmented Knowledge Sequence

Create a restrained architectural documentary-style visual sequence showing software architecture knowledge scattered across different evidence sources: source code, Infrastructure as Code, ADRs, diagrams, slide decks, monitoring dashboards and memory notes. The scene should not look like a product dashboard. It should feel like an editorial visualisation of incomplete architectural evidence. Use subtle motion potential: artefacts can drift, align and reveal partial outlines of a system.

## Journal Source Context

Create a calm close view of a canonical Markdown episode or rendered journal page, with highlighted fragments about understanding architecture. The image should feel grounded in a real repository or technical journal, not decorative stock footage. Leave space for small question fragments.

## Review Checks

- All visible artefact types are supported by Episode 0001.
- No unsupported dates, company names, technologies or metrics.
- Text is sparse and readable.
- Visuals support the narration rather than becoming a slide deck.

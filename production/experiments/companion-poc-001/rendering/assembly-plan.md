# Assembly Plan

## Rough Cut Inputs

- Approved or scratch narration from [`prototype-script.md`](../prototype-script.md).
- Scene model from [`scene-list.yaml`](../scene-list.yaml).
- Asset metadata from [`asset-register.yaml`](../asset-register.yaml).
- Companion direction B stills or motion test.
- Fragmented knowledge sequence.
- Concept diagram transformation.
- Source trace closing card.

## Assembly Steps

1. Create a 16:9 timeline targeting approximately 80 seconds.
2. Place narration first and use it as the timing spine.
3. Add companion opening for S001 with title and disclosure.
4. Transition to source view and question fragments for S002.
5. Build the fragmented knowledge visual sequence for S003.
6. Transform fragments into the central concept diagram for S004.
7. Return to companion for S005 and end on the canonical source path.
8. Add temporary captions from the approved script.
9. Export a review rough cut and record asset versions.

## Review Package

The review package should include:

- rough cut export;
- transcript or caption draft;
- asset register with generated versions;
- prompts used for generated visuals;
- reviewer scorecard;
- known deviations from the script or storyboard.

## Assembly Constraints To Record

- Manual edits required.
- Timing mismatches between narration and visual beats.
- Text readability issues.
- Companion consistency problems.
- Any unsupported visual claim introduced during generation.
